    impatient mode 的问题

    skewer-mode的作者有另一个包用于html预览–impatient-mode
    这个mode同样很诱人，能够直接实时预览你的html文件，甚至不用保存，十分诱人性能也好但是有一个问题，html中如果有js文件包含，当js文件修改的时候并不能刷新页面，我是个懒人，这么费尽的事情我当然不愿意做，所以才有下面的内容。

    更好的选择–browser-sync

    实际上在我一番探索后，emacs内部并没有很好的解决方案，几次搜索之后看上了这个有趣的项目browser-sync尝试之后发现这个包的确好用。安装过程也很简单。

    安装之后在命令行下启动就行

    这个命令会启动一个服务器监听css文件夹下*.css文件和 当前文件夹下 *.html的变化并打开浏览器预览，如果有变化会刷新浏览器。
    当然我们希望所有的文件都能被监听。

    这个命令可以监听当前文件夹下所有的文件包括文件夹中的，这样文件只要有修改，浏览器就会马上刷新。我们只需要在命令行中调用这个命令就行了，但是这样有点不方便，我们希望编辑器中能直接使用这个命令，对于emacser来说，添加一个函数就行了～

    好了，每次需要预览的时候`M-x browser-sync RET`就好了。

    添加firefox-developer-edition支持，并设置为默认浏览器

    不得不说firefox的开发者版本真香，个人觉得比chromium还好用，不过emacs中并不直接支持使用，有如下两个方案
    1. 设置系统的默认浏览器为firefox-developer-edition
    2. 修改emacs的默认配置

    emacs中配置默认的浏览器实际上是设置一个variable,browse-url-browser-function直接调用一个函数用于启动浏览器想要修改他也不难

    设置好之后使用browse-url-***相关的命令便可以用设置的浏览器打开网站或者文件，但是目前来看没有专门为firefox-dev设置一个函数，我设置为browse-url-mozilla后也没有用firefox-dev打开，chromium都有。。。
    于是决定自己写一个函数来用稍微修改了一下brow-url-firefox并且copy到了我的配置文件中

    这样就大功告成了！
